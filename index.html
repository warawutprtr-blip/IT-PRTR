🌐 ใช้ Claude Chat เมื่อ…

ทำงาน knowledge work, ต้องการแชร์ให้ทีมหลายคนใช้ร่วมกัน, ต้องการ context จากเอกสาร, ใช้บน browser/mobile

🖥️ ใช้ Cowork เมื่อ…

ต้องการให้ Claude เข้าถึง file system, ทำ multi-step task อัตโนมัติ, schedule งาน, ไม่ต้องเฝ้าตลอด

⚡ ใช้ Claude Code เมื่อ…

เขียนโค้ด, refactor, review PR, ทำงานใน terminal, ต้องการให้ทีม dev ใช้ standard เดียวกันผ่าน Git

⚠️ Cowork เป็น Research Preview — history เก็บในเครื่องเท่านั้น ยังไม่ sync ข้าม device และยังไม่รองรับการแชร์ให้ทีม (ณ มีนาคม 2026)

ติดตั้ง Claude Code

📟 terminal
# ติดตั้ง Claude Code (ต้องมี Node.js) $ npm install -g @anthropic-ai/claude-code # หรือใช้ผ่าน Claude Desktop App → แท็บ Code ✓ Claude Code installed successfully
2

เข้าไปใน project folder แล้วรัน /init

📟 terminal
$ cd my-project $ claude Claude Code started in my-project/ # พิมพ์ /init เพื่อให้ Claude scan codebase > /init Scanning project structure... ✓ Generated CLAUDE.md with project context
💡 รัน /init ทุกครั้งที่เริ่ม project ใหม่ — Claude จะ scan package.json, README, และ structure ให้อัตโนมัติ จากนั้นแก้ไข CLAUDE.md ตามความเป็นจริง
3

โครงสร้างไฟล์ที่แนะนำ

my-project/ ├── CLAUDE.md # ← instruction หลักของ project ├── .claude/ │ ├── commands/ # custom slash commands (/review, /deploy) │ └── settings.json # permissions config ├── src/ │ └── CLAUDE.md # ← instruction เฉพาะ src/ folder └── docs/ └── CLAUDE.md # ← instruction เฉพาะ docs/ folder
💡ใช้หลาย CLAUDE.md แบบ hierarchical — Claude โหลดทั้ง root และ sub-folder instruction พร้อมกัน เหมาะสำหรับ monorepo
4

Commit CLAUDE.md เข้า Git เสมอ

📟 terminal
$ git add CLAUDE.md .claude/ $ git commit -m "feat: add Claude Code configuration" ✓ ทีมทุกคนได้ใช้ context เดียวกัน

📚 Skills — ความรู้เฉพาะทางที่ Claude ดึงมาใช้อัตโนมัติ

Skills คืออะไร?

ไฟล์ Markdown ที่บอก Claude วิธีทำงานเฉพาะทาง เช่น writing style, brand guidelines, review checklist — Claude ดึงมาใช้ อัตโนมัติ เมื่อ task เกี่ยวข้อง

Built-in Skills ที่มีมาให้

📄 pdf  ·  📝 docx  ·  📊 xlsx
📽️ pptx  ·  🎨 canvas-design
โหลดอัตโนมัติเมื่อขอไฟล์ประเภทนั้น

วิธีติดตั้ง Custom Skill
1

เปิด Cowork → กด Customize (แถบซ้าย)

เมนู Customize รวม Skills, Connectors, Plugins ไว้ในที่เดียว

2

เลือก Skills → Add skill → วาง Markdown หรืออัปโหลดไฟล์

📚 ตัวอย่าง Custom Skill: writing-style.mdCowork
# Writing Style — FreshLife Brand ## Tone - เป็นกันเอง สดใส ไม่ formal - ใช้ active voice เสมอ ## Rules - ห้ามใช้คำว่า "leverage", "synergy" - Emoji ไม่เกิน 2 ตัวต่อ paragraph ## Output Length - Email: ≤150 คำ, Blog: 500-800 คำ

🤖 Sub-agents — AI หลายตัวทำงานขนานกัน

Sub-agent คืออะไร?

Claude ตัวหลักแบ่งงานให้ Sub-agents ทำพร้อมกัน เช่น ตัวหนึ่ง research, ตัวหนึ่งเขียน, ตัวหนึ่ง format

ทำงานยังไง?

• Claude วางแผนและแบ่ง task ให้เอง
• Sub-agents ทำงานใน VM แยกกัน
• ดู progress ได้ real-time ในหน้า Cowork
• หยุด/redirect ได้ทุกขั้นตอน

วิธีใช้ Sub-agents ใน Cowork
1

เขียน task ที่มีหลายงานย่อย หรือบอกให้แบ่งงาน

🖥️ ตัวอย่าง prompt ที่ trigger Sub-agentsCowork
# ตัวอย่าง 1 — task ซับซ้อน Claude แบ่งเอง ช่วยทำรายงาน Competitor Analysis สำหรับคู่แข่ง 5 ราย โดยดูข้อมูล website, pricing, และ features ของแต่ละราย แล้วสรุปลงใน Excel # ตัวอย่าง 2 — ขอให้แบ่งงานชัดเจน แบ่งงานให้ Agent หลายตัวทำพร้อมกัน: Agent 1 research ตลาด, Agent 2 เขียน summary, Agent 3 สร้าง slide
2

Claude วางแผนและแสดงขั้นตอนก่อนทำ — ตรวจและ Approve ได้

จะเห็น panel: แผนงาน → Sub-agents ที่รัน → progress แต่ละตัว — กด Pause เมื่อต้องการเปลี่ยนทิศทาง

3

รอให้ Claude รวมผลลัพธ์และ deliver ไฟล์สำเร็จรูปให้

💡Sub-agents เหมาะกับงานที่ "ทำได้พร้อมกัน" เช่น research หลายบริษัท, แปล+สรุป+format, สร้างหลายไฟล์ในครั้งเดียว

⚙️ Tips & Tricks เพิ่มเติม

เทคนิคเฉพาะของแต่ละเครื่องมือ — เลือก tab เพื่อดู tips ที่ตรงกับการใช้งานของคุณ

🔗 Connectors สำหรับ Claude Chat

Connectors เชื่อม Claude กับ tools ภายนอก — ตั้งครั้งเดียวใช้ได้กับทุก Project และทุก chat

📊 Google Drive

Claude อ่านและเขียน Doc, Sheet, Slide ได้โดยตรง — ไม่ต้อง copy-paste

💬 Slack

Claude ดึง context จาก channel หรือ post summary ลง channel ได้

📝 Notion

อ่านและสร้าง page ใน workspace ของทีมได้

📧 Gmail

Claude อ่าน draft และส่ง email ในนามคุณได้ผ่าน Connector

📅 Google Calendar

Claude ดู schedule และสร้าง event ในปฏิทินได้

🔧 MCP Servers

เชื่อมต่อ custom API และ database ของบริษัทผ่าน Model Context Protocol

วิธีติดตั้ง Connector บน Claude Chat
1

ไปที่ Settings → Connectors

คลิกรูปโปรไฟล์มุมขวาบน → Settings → เลือกแท็บ Connectors

2

เลือก Connector ที่ต้องการ แล้วกด Connect

ระบบจะขอให้ Authorize ผ่าน OAuth — ทำครั้งเดียว จากนั้น Claude เข้าถึงได้ทันที

3

เปิดใช้งานใน chat หรือ Project

ใน chat ให้กดไอคอน + แล้วเลือก Connector ที่ต้องการ — หรือพิมพ์ถามได้เลย เช่น "สรุปไฟล์ใน Drive ของฉัน"

🧠 Context Management — ยืดอายุ Session

• ใส่ข้อมูลสำคัญใน Project Knowledge แทนการ paste ใน chat
• แยก chat ตาม task — ไม่ใช้ chat เดียวทำทุกเรื่อง
• Pin project ที่ใช้บ่อยไว้ที่ sidebar เพื่อเข้าถึงเร็วขึ้น
• ใช้ Project Knowledge สำหรับข้อมูลที่เปลี่ยนน้อย เช่น Policy, SOP

📋 Checklist: ก่อนแชร์ Project/Agent ให้ทีม
✓

ทดสอบด้วยคำถามจริงอย่างน้อย 10 ข้อ

รวมถึง edge cases เช่น คำถามนอกเรื่อง หรือข้อมูลที่ไม่มีในเอกสาร

✓

ระบุ Owner 1 คนสำหรับ maintain

รับผิดชอบ update instruction และ knowledge base เมื่อข้อมูลเปลี่ยน

✓

เอกสารไฟล์ล่าสุดเท่านั้น

ลบ/แทนที่เวอร์ชันเก่า — Claude ใช้ทุกไฟล์ที่อัปโหลด ไฟล์เก่าอาจทำให้ตอบผิด

✓

แจ้งทีมถึงวิธีใช้งานที่ถูกต้อง

บอกทีมว่า Agent นี้ทำอะไร ไม่ทำอะไร และควรถามแบบไหนเพื่อผลลัพธ์ดีที่สุด

🔗 Connectors ใน Cowork — การติดตั้งและแนวทางการใช้งาน

Connectors ให้ Claude เข้าถึง apps และ services ภายนอกได้โดยตรง — ใน Cowork, Connector ทำงานร่วมกับ file system เพื่อ workflow อัตโนมัติที่ทรงพลัง

⬇️ วิธีติดตั้ง Connector ใน Cowork
1

เปิด Claude Desktop → Settings → Connectors

คลิกชื่อ Account ด้านล่างซ้าย → Settings → เลือก Connectors ในเมนูซ้าย

2

เลือก Connector ที่ต้องการ → กด "Connect"

จะมี popup ให้ Authorize ผ่าน OAuth — กด Allow เพื่อให้ Claude เชื่อมต่อ

3

เปิดใช้ Connector ใน Cowork session

ใน Cowork ให้กดไอคอน + แล้วเลือก Connector — หรือพิมพ์ถาม Claude ได้เลย Claude จะใช้ Connector อัตโนมัติ

4

ระบุ scope ใน Global Instructions (แนะนำ)

⚙️ Global Instructions — Connector guidanceCowork
## Connector Rules - ใช้ Google Drive เฉพาะ folder ~/Documents/Work/ เท่านั้น - ห้าม post ใน Slack โดยไม่ได้รับอนุมัติ — ให้ draft ไว้ก่อนแล้วขอ approve - แจ้งทุกครั้งเมื่อจะเขียน/แก้ไขไฟล์ผ่าน Connector
💡 แนวทางการใช้ Connector ใน Cowork ให้ได้ประโยชน์สูงสุด

📊 Google Drive + Cowork

ให้ Claude ดึงข้อมูลจาก Sheet → ประมวลผล → สร้างรายงาน .docx ใน local folder อัตโนมัติ

💬 Slack + Cowork

สั่งให้ Claude อ่าน Slack channel → สรุป action items → บันทึกเป็นไฟล์ → หรือ schedule ให้ทำทุกเช้า

📝 Notion + Cowork

Claude อ่าน task list จาก Notion page → ประมวลผล → update status กลับ Notion โดยอัตโนมัติ

🔧 MCP Server + Cowork

เชื่อม Claude กับ database หรือ internal API ของบริษัท — Claude query, process, แล้วสร้างรายงานใน local file system

⚠️ ระบุ permission ของ Connector ใน Global Instructions เสมอ — ป้องกัน Claude ทำงานเกินขอบเขตที่ต้องการ
🧩 Add Plugins ใน Cowork — การติดตั้งและแนวทางการใช้งาน

Plugin คือ package ที่รวม Connectors + Sub-agents + Instructions — ติดตั้งได้ในคลิกเดียว พร้อมใช้งานทันที

Plugin มีอะไรบ้าง?

🔗 Connectors — เชื่อมต่อ apps ที่ต้องการ
🤖 Sub-agents — AI เฉพาะทางสำหรับ task ต่างๆ
📋 Instructions — workflow + rules สำเร็จรูป
⚙️ Settings — ตั้งค่าพร้อมใช้ทันที

เหมาะกับใคร?

👤 ทีมที่ต้องการ workflow มาตรฐานเดียวกัน
🏢 องค์กรที่แชร์ Cowork config ให้สมาชิก
⚡ ผู้ที่ต้องการ setup เร็วโดยไม่ต้องตั้งค่าเอง

⬇️ วิธีติดตั้ง Plugin ใน Cowork
1

เปิด Claude Desktop → กด "Customize" หรือไอคอน ✨

อยู่ที่แถบซ้ายด้านล่าง — คลิกเพื่อเปิดหน้า Customize Claude

2

เลือกแท็บ "Plugins" → Browse หรือ Install from URL

หากทีมของคุณมี Plugin URL อยู่แล้ว ให้วาง URL แล้วกด Install

🧩 Plugins — Install from URL
# ตัวอย่าง URL ของ Plugin ที่แชร์ในองค์กร Plugin URL: https://your-org.com/cowork-plugins/daily-report.json → Installing: Daily Report Plugin v1.2... ✓ Plugin installed. 3 connectors, 2 sub-agents loaded.
3

Authorize Connectors ที่ Plugin ต้องการ

ถ้า Plugin ต้องการ Google Drive หรือ Slack ระบบจะแสดง OAuth popup ให้ Allow ตามลำดับ

4

เปิดใช้ Plugin ใน Cowork

ไปที่แท็บ Cowork → กดไอคอน + → เลือก Plugin ที่ติดตั้งแล้ว

💡 แนวทางการสร้างและใช้ Plugin ให้มีประสิทธิภาพ

🏗️ การออกแบบ Plugin ที่ดี

• 1 Plugin = 1 workflow หลัก — อย่าใส่ทุกอย่างใน Plugin เดียว
• ตั้งชื่อให้ชัดเจน เช่น "Daily-Report-v2" ไม่ใช่แค่ "Plugin1"
• ระบุ version และ changelog ใน Plugin description

🔄 การ update Plugin

• แก้ไข Plugin JSON แล้วส่ง URL ใหม่ให้ทีม
• หรือใช้ versioned URL เพื่อ rollback ได้ง่าย
• ทดสอบ Plugin ในเครื่องตัวเองก่อนแชร์ทีม

💡 Plugin เป็นวิธีที่ดีที่สุดสำหรับแชร์ Cowork workflow ให้ทีม — แชร์ URL เดียว ทุกคนได้ setup เดียวกัน

🔒 Permission Modes ใน Claude Code
Modeการทำงานเหมาะกับ
Defaultขอ permission ก่อนทำทุกอย่างใช้เรียนรู้ / project ใหม่
AcceptEditsauto-approve file changesงาน coding ปกติ
Plan Modeแสดงแผนก่อน ขอ approve แล้วค่อยทำtask ซับซ้อน >3 ไฟล์
BypassPermissionsข้ามทุก permission promptเฉพาะ trusted env ที่ปลอดภัยมาก
⚠️เริ่ม complex task ด้วย Plan Mode เสมอ — ให้ Claude แสดงแผนก่อน approve แล้วค่อยเปลี่ยนเป็น AcceptEdits เพื่อความเร็ว
